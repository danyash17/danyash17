- Medione is my biggest project so far. It is a healthcare CRM developed for Belarusian healthcare industry. It's a client-server application with monolithic backend and specific medical server. It implements basic features as authentication/registration (MFA included), profile settings, patient-doctor role division etc. Main business-valued system modules are visit booking system, e-medcard contaiment module, and document processing module. The last is based on FHIR standard, all entities are in FHIR resources forms. The killer feature is a developed connectivity with state drug registry (there is no public API for it, so I've made it scrapper-based) with a medical prescription generation function. The program has a specific flow that sets up prescription data using internationally acknowledged standards and terminologies, as SNOMED, UCUM, ICD and Orpanet. Generated prescription can be submitted both in digital and paper forms, it uses legally approved forms and may be used in existing state document exchange systems.
